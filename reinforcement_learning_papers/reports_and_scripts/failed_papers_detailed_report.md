## Root Cause Analysis

### Primary Issue: Context Window Overflow

**Tested paper:** `1605_07736.txt` (128KB)

**Error message:**
```
Error: Failed to call chat-completions api
No servers can handle a request of 55,444 tokens (contextSize + max_tokens too large).
```

### Technical Details

The Llama-3.3-70B-Instruct model has a context window limit. Papers extracted to text often exceed this limit:

- **Small PDFs** (~10-20 pages): ~50-100KB text → ~15-25K tokens ✓
- **Medium PDFs** (~20-40 pages): ~100-150KB text → ~25-40K tokens ✓  
- **Large PDFs** (~40-60 pages): ~150-200KB text → ~40-55K tokens ✗
- **Very Large PDFs** (60+ pages): >200KB text → >55K tokens ✗

**Conversion rate:** Approximately 1 byte ≈ 0.43 tokens for extracted PDF text.

### Solution Options

1. **Chunking Strategy**
   - Split large papers into sections (Introduction, Method, Experiments, Conclusion)
   - Process each chunk separately
   - Merge results

2. **Use Larger Context Models**
   - Models with 128K+ token context windows
   - Examples: Claude, GPT-4-Turbo, Llama-3-400B

3. **Pre-processing**
   - Extract only key sections (abstract, introduction, conclusion)
   - Remove references, appendices
   - Clean up formatting

4. **Manual Review**
   - For very important papers
   - Complex or ambiguous categorization
